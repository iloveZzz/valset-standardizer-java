# QLExpress 规则引擎落地任务计划

## 目标

将当前写死在 Java 代码里的解析和标准化判定逻辑，收敛为可配置、可发布、可回滚、可回放的 QLExpress 规则体系。

落地原则：

1. Java 保留文件读取、ODS 落库、任务编排、结果持久化。
2. QLExpress 负责模板选择、表头识别、行分类、字段映射、值转换、评分决策。
3. 规则必须版本化、可测试、可追踪、可灰度。
4. 现有行为先封装成默认版本，不允许一上来破坏现有解析结果。

---

## 当前进度

已完成的落地点：

1. 引入 `com.alibaba:qlexpress4` 依赖。
2. 新增 QLExpress 规则执行器骨架 `QlexpressParseRuleEngine`。
3. 将 `OdsValuationDataParser` 的表头识别、数据起始行、行分类接入规则执行器。
4. 将标准化层的表头映射切换到 `QlexpressHeaderMappingEngine`，并保留旧四步引擎作为回退。
5. 将标准化层的来源映射、指标兜底策略接入 QLExpress 规则表达式。
6. 新增模板与规则持久化对象。
7. 新增规则引擎 Liquibase 建表脚本。

后续继续推进时，优先完成：

1. 标准化链路的规则外置。
2. 规则上传 / 预览 / 发布接口。
3. 样例回归与表达式追踪联动。

## 二、详细任务拆解

### 阶段 0：现状基线冻结

#### 任务 0.1：梳理当前硬编码规则清单

目标：

- 列出当前所有仍写死在 Java 中的解析、分类、映射、归一化规则。
- 明确哪些规则必须保留为内置函数，哪些可以迁移到 QLExpress。

交付物：

- 规则清单文档
- 规则分层建议
- 内置函数候选列表

验收标准：

- 能从清单中定位到 `OdsValuationDataParser`、`ExcelParsingSupport`、`SubjectHierarchySupport`、`ExternalValuationStandardizationService` 的主要规则点。

---

#### 任务 0.2：补齐现有行为回归样例

目标：

- 为当前主流程整理一批样例文件和预期输出。
- 作为后续规则化改造的回归基线。

交付物：

- 样例文件集
- 预期 JSON 结果
- 断言脚本

验收标准：

- 规则改造前后，样例文件的主流程输出一致。

### 阶段 1：QLExpress 基础设施接入

#### 任务 1.1：引入 QLExpress 依赖

目标：

- 在工程中接入 `com.alibaba:qlexpress4`。
- 确认 JDK、Spring Boot、打包和测试链路可用。

交付物：

- Maven 依赖变更
- 基础启动验证

验收标准：

- 项目可编译。
- QLExpress 可以在单元测试中正常执行简单表达式。

---

#### 任务 1.2：实现规则执行器骨架

目标：

- 新增统一的规则执行入口。
- 封装表达式编译、缓存、执行、超时、异常处理。

建议接口：

- `ParseRuleEngine`
- `RuleExpressionService`
- `RuleCompileCache`

交付物：

- 执行器接口
- 默认执行器实现
- 基础单测

验收标准：

- 规则表达式可单独执行。
- 缓存可命中。
- 超时可生效。

---

#### 任务 1.3：实现安全策略与白名单

目标：

- 关闭脚本对任意 Java 类的自由访问。
- 仅暴露允许的函数与对象上下文。

交付物：

- 安全策略配置
- 函数白名单
- 类白名单

验收标准：

- 非白名单脚本访问被拒绝。
- 规则执行不依赖开放式 `import`。

验收标准：

- 模板可区分草稿、已发布、已禁用状态。

#### 任务 2.2：重构规则步骤表

- 新增 `t_file_parse_rule_step` 作为步骤表，保留旧 `t_file_parse_rule` 兼容历史字典数据。

交付物：

- 查询装配逻辑

验收标准：

- 字典数据仍可被标准化服务读取。

#### 任务 2.4：新增样例回归表与发布日志表

## 四、推荐实施顺序

1. 先做阶段 0，冻结现有行为。
2. 再做阶段 1 和阶段 2，补齐规则引擎和表结构。
3. 接着做阶段 3、4、5，把硬编码规则逐步外置。
4. 再补阶段 6、7，保证可发布、可追踪、可验收。
5. 最后做阶段 8，平滑切换默认模板。

---

## 五、完成定义

当下面条件全部满足时，说明 QLExpress 规则体系落地完成：

1. 新模板可以通过上传规则包完成，无需修改 Java 代码。
2. 规则可版本化、可发布、可回滚。
3. 规则执行有缓存、有超时、有安全边界。
4. 解析与标准化结果可追踪、可回放、可审计。
5. 现有主流程样例通过回归测试。
